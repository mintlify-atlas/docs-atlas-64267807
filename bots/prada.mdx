Prada is a Python Discord bot built on a custom `discord.py` fork (`rapptz/discord.py` installed from GitHub). Its architecture centres on a `Slut` subclass of `commands.AutoShardedBot` that discovers and loads cogs recursively from a `categories/` directory tree. Configuration lives in two files — `config.py` for the token and owner IDs, and `blacklist.json`/`whitelist.json` for access-control data.

<Warning>
  `config.py` contains a hardcoded bot token and owner ID list in plain text. The token in the repository should be considered compromised and must be replaced before running the bot.
</Warning>

## Entry point

`main.py` is minimal — it imports `Slut` from `core` and calls it directly:

```python
from core import Slut

if __name__ == "__main__":
    Slut()
```

The `Slut.__init__` constructor calls `self.run(token=config.token)` synchronously, so the bot starts as soon as the class is instantiated.

## Bot configuration (`core/slut.py`)

```python
class Slut(Bot, commands.AutoShardedBot):
    def __init__(self, *args, **kwargs) -> None:
        super().__init__(
            owner_ids=config.owner_ids,
            intents=Intents.all(),
            shard_count=1,
            command_prefix=self.get_prefix,
            help_command=HelpCommand(),
            case_insensitive=True,
            strip_after_prefix=True,
            allowed_mentions=AllowedMentions(
                replied_user=False,
                everyone=False,
                roles=False,
                users=True,
            ),
            status=Status.online,
        )
        self.run(token=config.token)
```

The prefix is hardcoded to `,` (comma) via `when_mentioned_or(",")`. Responding to the bot's direct mention also sends the configured prefix back to the user.

## Configuration files

### `config.py`

```python
token = "<BOT_TOKEN>"
owner_ids = [628236220392275969, 1040008141301088276, 193315346923388929]
```

### `blacklist.json`

Stores a flat list of blacklisted user IDs under `blacklist.users`. The `blacklist` cog (`categories/blacklist/blacklist.py`) reads and mutates this file.

### `whitelist.json`

```json
{"whitelist": {}, "is_enabled": true}
```

The `is_enabled` flag gates whitelist enforcement. When enabled, only whitelisted entities may invoke commands.

## Cog loading

Cog discovery happens in `on_connect` by walking all subdirectories of `./categories/`:

```python
async def load_cogs(self):
    for category_folder in os.listdir("./categories"):
        category_path = os.path.join("./categories", category_folder)
        if os.path.isdir(category_path):
            for file in os.listdir(category_path):
                if file.endswith(".py"):
                    cog_name = f"categories.{category_folder}.{file[:-3]}"
                    await self.load_extension(cog_name)
```

`jishaku` is loaded separately in `on_ready`.

## Rate limiting

The bot applies a shared cooldown at both the channel and user level (3 requests per 3 seconds per bucket) before dispatching any command:

```python
self.channel_cd = CooldownMapping.from_cooldown(3, 3, BucketType.channel)
self.user_cd    = CooldownMapping.from_cooldown(3, 3, BucketType.user)

async def on_message(self, message):
    if not message.author.bot and message.guild:
        if not self.cooldown(message):
            return await self.process_commands(message)
```
