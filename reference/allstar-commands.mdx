Allstar responds to prefix-based commands. The default prefix is set in `config.json` (`default_prefix`). Each guild can override it with `prefix set <char>`, and individual users can set their own personal prefix with `selfprefix`. You can also mention the bot directly in place of the prefix. Commands have a built-in 3.5-second cooldown per user ‚Äî reacting with ‚åõ means you triggered it too quickly.

<Note>
  The `antinuke`, `antispam`, `antialt`, `antilink`, and `antivanity` commands are restricted to the server owner. Most other moderation commands require `MANAGE_GUILD` or the relevant Discord permission.
</Note>

## Command structure

Every command file exports the same module shape. Here is `kick.js` as a representative example:

```javascript
const { MessageEmbed, Permissions } = require('discord.js');
const { default_prefix, color, error, owner, checked, xmark } = require("../config.json");
const talkedRecently = new Set();

  name: 'kick',
  description: 'kick mentioned user from the server',
  aliases: [],
  usage: '```YAML\n\n kick @heist#0001 spamming ``` ',
  category: "moderation",
  guildOnly: false,
  args: false,
  permissions: {
    bot: [],
    user: [],
  },
  execute: async (message, args, client) => {
    // Rate-limit check: 3.5 s cooldown per user
    if (talkedRecently.has(message.author.id)) {
      message.react('‚åõ');
    } else {
      if (!message.member.permissions.has([Permissions.FLAGS.KICK_MEMBERS]))
        return message.reply({ embeds: [missperms] });

      let mentionedMember =
        await message.mentions.members.first() ||
        message.guild.members.cache.get(args[0]) ||
        message.guild.members.cache.find(r =>
          r.user.username.toLowerCase() === args.join(' ').toLocaleLowerCase()
        );

        message.reply({ embeds: [banned] })

```

All 119 command files follow this pattern. The `talkedRecently` `Set` enforces the per-user cooldown. Member resolution accepts a mention, user ID, username, or display name.
